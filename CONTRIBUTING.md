# 参与共建 Contributing

欢迎参与流向 FlowSign 的共建。这是一个早期项目，欢迎各种形式的贡献。

---

## 共建方向

| 方向 | 说明 |
|------|------|
| 产品 / UX | 用户路径、交互设计、结果页结构 |
| 文案 / 内容 | 8种流向文案、题库扩充、分享卡文案 |
| 前端开发 | 小程序 / H5 / Web |
| 心理学 / 行为科学 | 维度设计、测量有效性、题目优化 |
| 传播 / 运营 | 小红书验证、内容策略 |

---

## 参与方式

### 提 Issue
- 发现逻辑问题、设计漏洞、文案问题 → 开 Issue 说明
- 有新想法或建议 → 开 Issue 讨论，不要直接 PR

### 提 PR
- 基于已讨论的 Issue 进行修改
- 每个 PR 只做一件事，不要混合多个改动
- PR 标题格式：`[类型] 简短描述`
  - 类型：`docs` / `design` / `copy` / `logic` / `fix`
  - 示例：`[copy] 补充散射流结果页文案`

### 分支命名
```
feature/描述    新内容或功能
fix/描述        修正问题
docs/描述       文档更新
```

## AI 分析日志

本项目使用 AI（Claude Code）辅助系统设计和逻辑审查。

所有 AI 的判断和建议统一记录在 Issue **#1 [AI分析日志]** 中，前缀标注 `[CC]`。

这些内容代表 AI 的分析视角，不是最终决策，供共建者参考和讨论。

## 基本原则

- 没有坏想法，只有没说清楚的想法
- 先讨论再动手，避免无效 PR
- 文案改动需要说明改动理由
- 逻辑类改动需要说明原问题是什么
